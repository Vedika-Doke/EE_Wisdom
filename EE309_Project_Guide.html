01 Project Overview

This project demonstrates how non-linear circuits create new frequencies that weren't in the original signal. We generate a clean two-tone signal using a microcontroller + DAC, feed it through a non-linear element (diode), and then measure the new frequencies that appear using an ADC + digital signal processing.

What We Build

  1. Signal Generator: The AT89C5131A microcontroller computes sine values in real-time using an NCO (Numerically Controlled Oscillator) and outputs them through the MCP4921 DAC at 16,384 samples/second. The analog output is a clean sum of two cosines: x(t) = A[cos(2π·410·t) + cos(2π·470·t)]
  2. Non-Linear Distortion: The clean signal passes through a 1N4148 diode, which has an exponential I-V curve. This non-linearity multiplies the signal with itself, creating new frequency components (intermodulation products).
  3. Measurement: The MCP3008 ADC samples the distorted signal. The microcontroller performs coherent I/Q detection (essentially a single-bin DFT) to measure the amplitude of each frequency of interest.

02 Theory — Why Non-Linearity Creates New Frequencies

A linear system only scales and time-shifts signals. If you put in two frequencies, you get out the same two frequencies (maybe with different amplitudes and phases). No new frequencies appear.

A non-linear system (like a diode) effectively multiplies the signal with itself. When you multiply two cosines, trigonometric identities create sum and difference frequencies:

cos(A) · cos(B) = ½[cos(A-B) + cos(A+B)]

This is how new frequencies appear from a non-linear element — the squaring and cubing operations generate all possible sums and differences of the input frequencies.

Non-Linear Response Model

A weakly non-linear system has the input-output relationship:

y(t) = α1·x(t) + α2·x2(t) + α3·x3(t)

Where α1 is the linear gain, α2 is the 2nd-order non-linearity, and α3 is the 3rd-order non-linearity.

For Two-Tone Input x(t) = A[cos(2πf1t) + cos(2πf2t)]

2nd-order products (from α2·x2):

DC,   f2 − f1 = 60 Hz,   f1 + f2 = 880 Hz,   2f1 = 820 Hz,   2f2 = 940 Hz

3rd-order products (from α3·x3):

2f1 − f2 = 350 Hz,   2f2 − f1 = 530 Hz,   3f1 = 1230 Hz,   3f2 = 1410 Hz

We focus on measuring amplitudes at four frequencies: 60 Hz, 350 Hz, 410 Hz, 470 Hz.

The 60 Hz and 350 Hz components don't exist in the original signal at all — they are entirely created by the diode's non-linearity. Their amplitudes are proportional to α2A2 and α3A3 respectively, telling us how non-linear the circuit is.

Why P1.1 for DAC CS? P1.1 is the hardware SS (Slave Select) pin on the AT89C5131A. In master mode, the hardware SPI block can automatically drive SS — but since we have TWO slaves (DAC and ADC), we need manual control. Why P1.0 for ADC CS? It's the next available GPIO on Port 1 that isn't used by SPI or other peripherals. If your board has P1.0 occupied, change one line in mcp3008.c: sbit ADC_CS = P1^0; to whatever pin you use.

SPI Bus Topology

SPI is a bus — three signal wires are shared among all devices. Only the CS (chip-select) line is unique per device. At any moment, exactly ONE device has its CS pulled LOW and is listening. The others ignore the bus.

Pt-51 (AT89C5131A) +-------------------+ | | | P1.0 (ADC_CS) ---+--[wire]---> MCP3008 pin 10 (CS) | P1.1 (DAC_CS) ---+--[wire]---> MCP4921 pin 2 (CS) | | | P1.5 (MISO) -------- bus ------> MCP3008 pin 12 (DOUT) | | | P1.6 (SCK) -------- bus --+--> MCP4921 pin 3 (SCK) | | +--> MCP3008 pin 13 (CLK) | | | P1.7 (MOSI) -------- bus --+--> MCP4921 pin 4 (SDI) | | +--> MCP3008 pin 11 (DIN) +-------------------+
How bus sharing works at the hardware level: When DAC_CS = 0 and ADC_CS = 1, the MCP4921 is listening and the MCP3008 ignores the bus (its DOUT is high-impedance). When we flip to DAC_CS = 1 and ADC_CS = 0, the MCP3008 wakes up and the MCP4921 goes to sleep. The clock (SCK) and data (MOSI) wires physically go to both chips simultaneously, but only the chip with CS=LOW processes the bits.

06 Non-Linear Circuit

Topology: Series Diode with Pull-Down Resistor

This is the simplest circuit that produces usable intermodulation products. Two components: one 1N4148 diode and one 10 kΩ resistor.

MCP4921 MCP3008 Pin 7 (VOUT) Pin 1 (CH0) | | +---[ 1N4148 diode ]---+--- node A ---+--- Vout ----------------+ (anode →) | | (band = cathode)| [R = 10 kΩ] | | | GND | | ↑↑↑↑↑↑↑↑↑ | SCOPE PROBE HERE | for Phase 3 testing | (probe tip on node A, | ground clip on GND) Scope probe locations for each phase: Phase 1-2: Probe on MCP4921 Pin 7 (VOUT) — clean sine / two-tone beat Phase 3: Probe on node A (wire between diode cathode + R + MCP3008 CH0) — clipped waveform FFT mode: Same as Phase 3 probe point — switch scope to Math/FFT
How the diode creates non-linearity:

A diode's current-voltage relationship is I = Is(eV/nVT - 1), where VT ≈ 26 mV at room temperature. This exponential curve means:
  • Below ~0.5V: diode is nearly OFF. Very little current flows. Output ≈ 0V (pulled to GND by R).
  • Around 0.6V: the "knee" region. Current increases rapidly. This is where the strongest non-linearity lives.
  • Above ~0.7V: diode is fully ON. Behaves almost linearly (Vout ≈ Vin - 0.7V).
The transition region around 0.5–0.7V is where the squaring and cubing happen. When the two-tone signal passes through this region, the intermodulation products are generated.

Our DAC output swings 0–5V. In two-tone mode with half-amplitude per tone, the signal swings roughly 1.25V–3.75V centered at 2.5V. Most of this range is above the diode threshold, so the diode is ON for most of the cycle. The non-linearity appears at the bottom of the swing where the signal dips toward 0.6V.

Diode Orientation

The 1N4148 has a band printed on one end — that's the cathode. Wire it so:

  • Anode (no band) → connects to DAC VOUT
  • Cathode (band) → connects to node A (R + ADC input)

07 SPI Library

What is SPI?

SPI (Serial Peripheral Interface) is a synchronous serial protocol with 4 signals:

  • SCK (Serial Clock): master generates clock pulses
  • MOSI (Master Out Slave In): data from master to slave
  • MISO (Master In Slave Out): data from slave to master
  • CS (Chip Select): active-low, selects which slave is active

Data is shifted out MSB-first on each clock edge. Both MCP4921 and MCP3008 use SPI Mode 0,0 (CPOL=0, CPHA=0): clock idles LOW, data is sampled on the RISING edge.

AT89C5131A Hardware SPI Registers

Instead of bit-banging (manually toggling GPIO pins), we use the chip's built-in SPI hardware. This is much faster (~3 μs per byte vs ~180 μs for bit-bang) and frees the CPU to do other work while the SPI transfer happens.

spi.c

#include <at89c5131A.h>
#include "spi.h"

/* AT89C5131A SPI SFRs — at89c5131A.h should already declare these.
   The #ifndef guards are a safety net in case it doesn't. */
#ifndef SPCON
sfr SPCON = 0xC3;
sfr SPSTA = 0xC4;
sfr SPDAT = 0xC5;
#endif

#define SPCON_CFG 0x71  /* SPEN=1, SSDIS=1, MSTR=1, mode 0,0, Fosc/8 */

void spi_init(void) {
    unsigned char dummy;
    SPCON = SPCON_CFG;
    dummy = SPSTA;   /* Clear any stale SPIF flag from power-on.
                        Without this, the first spi_xfer() call could
                        see SPIF=1 from a previous state and return
                        immediately with garbage data. The two-step
                        clear sequence: read SPSTA, then read SPDAT. */
    dummy = SPDAT;
}

unsigned char spi_xfer(unsigned char b) {
    /* Write byte to SPDAT to start transfer.
       The hardware shifts out our byte on MOSI while simultaneously
       shifting in the slave's response on MISO. */
    SPDAT = b;

    /* Poll SPIF (bit 7 of SPSTA) until the 8-bit transfer completes.
       At 3 MHz SCK, this takes 8/3M = 2.67 us per byte.
       Reading SPSTA when SPIF=1, then accessing SPDAT, clears the flag. */
    while (!(SPSTA & 0x80));

    /* Return the byte that was shifted in from the slave. */
    return SPDAT;
}

08 DAC Driver (MCP4921)

How the MCP4921 Works

The MCP4921 converts a 12-bit digital number (0–4095) into an analog voltage (0–VREF). You send it a 16-bit SPI word: 4 config bits + 12 data bits. On the rising edge of CS, the DAC latches the new value and VOUT changes to the corresponding voltage.

VOUT = VREF × D / 4096 = 5V × D / 4096
Example: D = 2048 → VOUT = 2.500 V (midpoint)
Example: D = 4080 → VOUT = 4.980 V (near max)

16-Bit Write Frame

Bit 15Bit 14Bit 13Bit 12Bits 11–0
0 = DAC A
(MCP4921 has only one channel)		 1 = Buffered VREF
(better linearity)		 1 = Gain 1×
(output = VREF × D/4096)		 1 = Active
(0 = shutdown mode)		 D[11:0]
12-bit data value

Config nibble = 0b0111 = 0x7. Full word = 0x7000 | (data_value & 0x0FFF).

Timing Diagram (one write cycle)

CS ____/\______________/\____ (LOW during transfer, HIGH to latch) SCK ____/\_/\_/\_/\_/\_/\_/\_/\_____ (16 clock pulses) MOSI ---[b15 b14 ... b1 b0]--- (16 data bits, MSB first) VOUT ........................XXXX (changes on CS rising edge)

mcp4921.c

#include <at89c5131A.h>
#include "spi.h"
#include "mcp4921.h"

sbit DAC_CS = P1^1;   /* SS pin on the SPI header */

#define DAC_CMD 0x7000  /* bits 15-12: 0_1_1_1 = DAC_A, buffered, 1x, active */

void dac_init(void) {
    DAC_CS = 1;         /* CS HIGH = DAC is idle / deselected */
}

void dac_write(unsigned int val12) {
    /* Build the 16-bit word: config nibble + 12-bit data */
    unsigned int word = DAC_CMD | (val12 & 0x0FFF);

    DAC_CS = 0;                                /* select DAC */
    spi_xfer((unsigned char)(word >> 8));     /* send high byte (config + D11..D8) */
    spi_xfer((unsigned char)(word & 0xFF));   /* send low byte (D7..D0) */
    DAC_CS = 1;                                /* deselect → DAC latches value, VOUT updates */
}

09 ADC Driver (MCP3008)

How the MCP3008 Works

The MCP3008 converts an analog voltage (0–VREF) into a 10-bit digital number (0–1023). You tell it which channel to read via a 3-byte SPI frame, and it sends back the result in the same frame.

Digital value = 1024 × VIN / VREF = 1024 × VIN / 5V
Example: VIN = 2.500 V → result = 512
Resolution: 5V / 1024 = 4.88 mV per step

3-Byte SPI Read Frame

Byte #What MCU Sends (MOSI)What ADC Returns (MISO)Explanation
1 0x01 don't care Start bit. The ADC starts its conversion when it sees a 1 after CS goes low.
2 0x80 | (ch << 4) xxxx_xx D9 D8 Bit 7 = 1 means single-ended (not differential). Bits 6:4 = channel number (0–7). ADC starts returning result bits.
3 0x00 (dummy) D7 D6 D5 D4 D3 D2 D1 D0 Remaining 8 bits of the 10-bit result. We clock out zeros just to receive these bits.

10-bit result = ((byte2 & 0x03) << 8) | byte3

mcp3008.c

#include <at89c5131A.h>
#include "spi.h"
#include "mcp3008.h"

sbit ADC_CS = P1^0;   /* change this if you wire CS to a different pin */

void adc_init(void) {
    ADC_CS = 1;         /* CS HIGH = ADC is idle / in shutdown */
}

unsigned int adc_read(unsigned char channel) {
    unsigned char hi, lo;

    ADC_CS = 0;                                              /* wake up ADC */
    spi_xfer(0x01);                                        /* byte 1: start bit */
    hi = spi_xfer((unsigned char)(0x80 | ((channel & 0x07) << 4)));  /* byte 2: SGL + channel */
    lo = spi_xfer(0x00);                                    /* byte 3: clock out result */
    ADC_CS = 1;                                              /* done, ADC goes to sleep */

    /* Extract 10-bit result: bits[1:0] of byte2 ++ all of byte3 */
    return ((unsigned int)(hi & 0x03) << 8) | lo;
}

10 Signal Generation (NCO)

What is an NCO?

A Numerically Controlled Oscillator is a digital technique for generating sine waves at any frequency from a single lookup table. Instead of storing a separate table for each frequency, you store ONE table (one full sine cycle) and step through it at different speeds.

Analogy: Imagine a clock face with 32 marks around the edge, each representing a sine value. A pointer starts at 12 o'clock and advances by a fixed angle each tick:
  • Small angle per tick → slow traversal → low frequency output
  • Large angle per tick → fast traversal → high frequency output
The "angle" is stored as a 16-bit integer (0–65535 represents 0–360°). The "step size" is called the phase increment (Δφ). On every timer tick, we add Δφ to the accumulator and use the top bits to index the sine table.

NCO Math

Δφ = round(65536 × fout / Fs)

For Fs = 16384 Hz:  Δφ = 4 × fout  (exact integer — no rounding error!)

f1 = 410 Hz → Δφ1 = 1640
f2 = 470 Hz → Δφ2 = 1880
Why Fs = 16384 Hz? Because 65536 / 16384 = 4, which means Δφ = 4 × f, always an exact integer for integer Hz values. No rounding error means the output frequency is exactly what we want (410.000 Hz, not 409.87 Hz).

Also, at 24 MHz crystal (2 MHz machine clock), Fs = 16384 requires the timer to fire every 122 machine cycles — leaving ~50 cycles of headroom for the ISR to execute.

   Center = 128, amplitude = 127, range = 1..255 */
code unsigned char sine32[32] = {
    128, 153, 177, 199, 218, 234, 245, 253,   /* rising */
    255, 253, 245, 234, 218, 199, 177, 153,   /* falling from peak */
    128, 103,  79,  57,  38,  22,  11,   3,   /* falling below center */
      1,   3,  11,  22,  38,  57,  79, 103    /* rising back to center */
};

LUT Indexing Optimization

To get a 5-bit index (0–31) from the 16-bit phase accumulator, we want bits 15–11. The naive way is phase >> 11, but on the 8051 this compiles to 11 individual right-shift operations on a 16-bit value (~22 machine cycles). Instead:

/* Fast: extract high byte, then shift right by 3 */
index = ((unsigned char)(phase >> 8)) >> 3;

/* This is equivalent to phase >> 11, but:
   - (phase >> 8) is just taking the high byte (free on 8051)
   - >> 3 is only 3 shifts of an 8-bit value (~3 cycles)
   Total: ~5 cycles instead of ~22 */

Two-Tone Summation

Each LUT value is 0–255, centered at 128.
Sum of two: 0–510, centered at 256.
Average (divide by 2): 0–255, centered at 128. No clipping.
Scale to 12-bit DAC: multiply by 16 (<< 4).
Combined: (s1 + s2) / 2 × 16 = (s1 + s2) << 3
Result range: 0–4080 out of 4095. Uses ~99.6% of DAC range.

11 Tone Amplitude Measurement

The Problem

After the non-linear circuit, the signal contains many frequencies mixed together. We need to extract the amplitude of specific tones (60, 350, 410, 470 Hz) from this messy signal. This is what the FFT does on the scope — but we need to do it digitally on the 8051 using ADC samples.

The Solution: I/Q Coherent Detection

This is essentially a digital lock-in amplifier or a single-bin DFT. For each frequency we care about, we multiply the signal by cos and sin at that frequency, then sum over many samples. If the signal contains that frequency, the sum grows large. If not, it averages to zero.

Step by step for one frequency fk:
  1. Sample the signal N times at rate Fs
  2. For each sample x[n], multiply by cos(2πfkn/Fs) and accumulate → Ik
  3. For each sample x[n], multiply by sin(2πfkn/Fs) and accumulate → Qk
  4. Amplitude = (2/N) × √(Ik2 + Qk2)
Why both cos and sin? Because the signal's phase is unknown. If the signal happens to be sin-aligned, the cosine correlation gives zero and vice versa. Using both and combining with √(I²+Q²) gives the amplitude regardless of phase.

Measurement Parameters

ParameterValueWhy This Value
Fs (ADC rate) 4,800 Hz Well above Nyquist for 470 Hz. All 4 target frequencies fit integer cycles in the window (critical for accuracy).
N (window) 480 samples Window duration = 100 ms. Integer cycles: 60×100ms=6, 350×100ms=35, 410×100ms=41, 470×100ms=47.
Why integer cycles matter: If the window doesn't contain an exact integer number of cycles of fk, the correlation "leaks" — a non-zero result appears even when the tone isn't present. With integer cycles, the orthogonality of sines guarantees clean separation between frequencies.
Phase 5 code will be added here once Phases 1–3 are verified on the scope. The implementation uses precomputed 8-bit cos/sin tables (one per target frequency), 16-bit multiply-accumulate in the ISR, and 32-bit magnitude computation in main().

main.c (Phase 1 & 2: Signal Generation)

#include <at89c5131A.h>
#include "spi.h"
#include "mcp4921.h"
#include "mcp3008.h"

/* ======================================================================
   Mode: 0 = single tone (410 Hz), 1 = two tones (410 + 470 Hz)
   Start with 0 to verify on scope, then change to 1 and rebuild.
   ====================================================================== */
#define TWO_TONE_MODE  0

#define TH0_RELOAD    0xFF
#define TL0_RELOAD    0x86    /* 65536 - 122 = 0xFF86 */
#define PHASE_INC_F1  1640u   /* 4 × 410 Hz */
#define PHASE_INC_F2  1880u   /* 4 × 470 Hz */

/* CKCON controls X2 mode. Declare at file scope so it compiles even
   if at89c5131A.h uses a different name. Same-address duplicate is OK in Keil. */
sfr CKCON_REG = 0x8F;

code unsigned char sine32[32] = {
    128, 153, 177, 199, 218, 234, 245, 253,
    255, 253, 245, 234, 218, 199, 177, 153,
    128, 103,  79,  57,  38,  22,  11,   3,
      1,   3,  11,  22,  38,  57,  79, 103
};

volatile unsigned int phase1 = 0;
volatile unsigned int phase2 = 0;

void timer0_isr(void) interrupt 1 {
    unsigned char s1;
#if TWO_TONE_MODE
    unsigned char s2;
    unsigned int  sum;
#endif

    /* Reload timer immediately to minimize jitter */
    TH0 = TH0_RELOAD;
    TL0 = TL0_RELOAD;

    /* Advance phase accumulator and look up sine value */
    phase1 += PHASE_INC_F1;
    s1 = sine32[((unsigned char)(phase1 >> 8)) >> 3];

#if TWO_TONE_MODE
    phase2 += PHASE_INC_F2;
    s2 = sine32[((unsigned char)(phase2 >> 8)) >> 3];
    /* Average two tones, scale to 12-bit DAC */
    sum = ((unsigned int)s1 + (unsigned int)s2) << 3;
    dac_write(sum);
#else
    /* Single tone, full amplitude, scale 8-bit to 12-bit */
    dac_write((unsigned int)s1 << 4);
#endif
}

void main(void) {
    /* Force standard 8051 timer clock (XTAL/12).
       AT89C5131A supports X2 mode (XTAL/6). If Pt-51 bootloader
       enables X2, all timer rates double and frequencies are wrong.
       CKCON (0x8F) bit 0 = T0X2. Clear it. */
    CKCON_REG &= ~0x01;

    spi_init();
    dac_init();
    adc_init();

    /* Timer 0: mode 1 (16-bit, manual reload).
       Preserve Timer 1 config in upper nibble (may be used by bootloader). */
    TMOD = (TMOD & 0xF0) | 0x01;
    TH0  = TH0_RELOAD;
    TL0  = TL0_RELOAD;
    ET0  = 1;    /* enable Timer 0 interrupt */
    EA   = 1;    /* global interrupt enable */
    TR0  = 1;    /* start Timer 0 */

    while (1);   /* ISR does all the work */
}

13 Code Audit — Bugs Found & Fixed

A rigorous line-by-line audit was performed on all source files, checking SPI protocol correctness, timer math, NCO phase math, DAC/ADC frame formats, 8051 timing budgets, and physical constraints. Here are the findings:

Bugs Fixed

#IssueImpactFix
1 CKCON X2 mode — AT89C5131A defaults to T0X2=1 after reset, meaning Timer 0 runs at XTAL/4 (6 MHz) instead of XTAL/12 (2 MHz). Output frequency would be 3× too fast. Wrong frequency, ISR overrun CKCON_REG &= ~0x01; at startup. Declared as sfr CKCON_REG = 0x8F; at file scope to avoid name conflicts with device header.
2 SPIF stale at power-on — if the SPIF flag is set from a previous state or power-on glitch, the first spi_xfer() call would see SPIF=1 immediately, skip the wait loop, and return stale data. The DAC would receive a corrupted first write. Garbage first DAC write Added dummy SPSTA + SPDAT reads in spi_init() to clear any stale SPIF via the two-step clear sequence.
3 Slow phase shift — phase >> 11 compiles to 11 individual right-shift operations on a 16-bit variable (~22 machine cycles on 8051). ISR ~17 cycles slower than needed Changed to ((unsigned char)(phase >> 8)) >> 3 — byte extract + 3 shifts (~5 cycles). Mathematically equivalent.
4 TMOD overwrote Timer 1 — TMOD = 0x01 zeroed the upper nibble, potentially breaking Timer 1 if the bootloader configured it (e.g., for UART). Possible bootloader conflict Changed to TMOD = (TMOD & 0xF0) | 0x01; to preserve Timer 1 config.

Timer reload race (TH0 written before TL0) ~2-4 cycle jitter per ISR, ~2% frequency offset TL0 is at ~0x03-0x08 when ISR fires, nowhere near 0xFF. The catastrophic case (TL0 wrapping during the write) is physically impossible within the 61 μs window. The systematic offset is stable and inaudible.
8-bit effective DAC resolution (32-entry LUT × 8-bit values into 12-bit DAC) Quantization harmonics above 6 kHz in the spectrum Our frequencies of interest (60–470 Hz) are well below the quantization floor. The harmonics are invisible in the scope FFT at our frequency range.
Two-tone ISR timing is tight (~95-120 cycles vs 122 budget) Occasional overrun could cause jitter Hardware SPI dominates the ISR (~10 cycles for 2 bytes at 3 MHz). The math adds ~24 cycles for the second tone. Total fits within 122 with ~10-20 cycles margin. If it occasionally overruns, the next interrupt fires immediately — stable but slightly lower effective sample rate.

Verified Correct

  • SPCON = 0x71 — AT89C5131A layout: SPR2(0), SPEN(1), SSDIS(1), MSTR(1), CPOL(0), CPHA(0), SPR1(0), SPR0(1). Mode 0,0 correct for MCP4921 and MCP3008. Fosc/8 = 3 MHz, within MCP3008 max (3.6 MHz @5V).

  • MCP3008 frame: 0x01, 0x80|ch<<4, 0x00 — start bit, single-ended + channel, dummy. Result extraction (hi&0x03)<<8|lo correct.
  • Sine LUT — all 32 values verified against 128 + 127×sin(2πi/32).
  • NCO math — phase increments 1640 and 1880 give exactly 410.0 and 470.0 Hz at Fs=16384.
  • Timer reload — 65536 − 122 = 65414 = 0xFF86. At 2 MHz machine clock, period = 61 μs.
  • SPIF clearing — spi_xfer reads SPSTA (polling), then returns SPDAT. Two-step clear matches AT89C5131A spec.

14 Debugging Guide

Step-by-Step Verification

Work through these in order. Don't skip ahead — each step depends on the previous one.

#TestHowExpectedIf It Fails
1 CPU running? Toggle an LED in while(1) with a delay loop LED blinks at ~1 Hz Programmer/flash failed. Check USB cable, FLIP software, chip power.
2 Timer ISR firing? Inside the ISR, increment a counter; toggle LED every 16384 ticks LED blinks at 1 Hz (16384 Hz / 16384 = 1 Hz) Check TMOD=0x01, ET0=1, EA=1, TR0=1. Also check CKCON — if X2 mode is on, the blink rate doubles.
3 SPI clock active? Probe P1.6 (SCK) on scope, 10 μs/div Bursts of 16 clock pulses every ~61 μs SPCON value wrong, or the SFR addresses (0xC3/0xC4/0xC5) don't match your chip's header.
4 CS toggling? Probe P1.1 (DAC CS) on scope, 10 μs/div Short LOW pulses (~6 μs wide) every ~61 μs Check sbit DAC_CS = P1^1; is correct.
5 DAC VOUT working? Probe MCP4921 pin 7 (VOUT), 1 V/div, 1 ms/div, DC coupling Clean 410 Hz sine wave, 0–5 V swing Check VREF (+5V), LDAC (GND), decoupling cap. If voltage is 10x too small, check probe/scope attenuation match.
6 Diode output? Probe node A (junction of diode cathode + resistor + MCP3008 CH0) Same sine but clipped below ~0.6 V (flat bottom) Check diode orientation: band (cathode) faces AWAY from DAC, toward node A.

Common Pitfalls

  • Probe attenuation mismatch: Scope probe has a switch (1×/10×) near the BNC end. The scope channel menu must match. If mismatched, amplitude reads 10× wrong. Always use 10× probe + 10× scope setting for these voltages.
  • Wrong pin probed: MCP3008 pin 12 = DOUT (digital SPI data), NOT the analog signal. Pin 1 (CH0) has the analog signal. MCP4921 pin 7 (VOUT) has the DAC output.
  • AC coupling on scope: Removes the 2.5 V DC offset, making the sine look smaller and centered at 0 V. Use DC coupling to see the true signal level.
  • Missing CKCON fix: If the Pt-51 bootloader enables X2 mode, Timer 0 runs at XTAL/6 instead of XTAL/12. All frequencies double. Fix: CKCON &= ~0x01; at the start of main().
  • Missing decoupling cap: SPI data gets corrupted by power supply noise. DAC outputs random voltages instead of smooth sine. Add 100 nF ceramic between VDD and GND on each chip, placed within 1 cm of the chip.
  • LDAC floating: If MCP4921 pin 5 (LDAC) isn't tied to GND, the DAC output updates at random times instead of on CS rising edge. Result: noisy, jittery waveform.
  • Slow bit-shift on 8051: phase >> 11 compiles to 11 shifts of a 16-bit value (~22 machine cycles). Use ((unsigned char)(phase >> 8)) >> 3 instead (~5 cycles). This can make the difference between the ISR fitting in the timer period or not.
