Probability — Formula Sheet Core Theory

Basic Rules

Addition Rule
P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
For mutually exclusive events: P(A ∩ B) = 0, so P(A ∪ B) = P(A) + P(B)
Multiplication Rule
P(A ∩ B) = P(A) · P(B|A) = P(B) · P(A|B)
For independent events: P(A ∩ B) = P(A) · P(B)
Conditional Probability
P(A|B) = P(A ∩ B) / P(B) P(B|A) = P(A ∩ B) / P(A)
Read: "probability of A given B has occurred"
Complement Rule
P(A') = 1 − P(A)
Use "at least one" problems: P(≥1) = 1 − P(none)

Counting & Combinatorics

Permutation
ⁿPᵣ = n! / (n−r)!
Order matters. Arranging r items from n distinct items.
Combination
ⁿCᵣ = n! / [r!(n−r)!]
Order does not matter. Choosing r items from n.
Circular Permutation
(n − 1)!
Arranging n people around a circular table (one fixed reference). Necklace/bracelet: (n−1)!/2
Repetition Allowed
nʳ
Choosing r items from n with replacement (order matters).
Identical Objects
n! / (n₁! × n₂! × ··· × nₖ!)
Arrangements of n objects with repetitions n₁, n₂,… of each type.
Stars & Bars
ⁿ⁺ᵣ⁻¹Cᵣ
Distributing r identical items into n distinct bins (zeros allowed). Without zeros: ⁿ⁻¹Cᵣ₋₁

Bayes' Theorem & Total Probability

Law of Total Probability
P(A) = Σ P(A|Bᵢ) · P(Bᵢ) for i = 1…n
Requires B₁, B₂,…Bₙ to be exhaustive & mutually exclusive (partition of sample space).
Bayes' Theorem
P(Bᵢ|A) = P(A|Bᵢ) · P(Bᵢ) / Σ P(A|Bⱼ)·P(Bⱼ)
Prior → Posterior. Used to "update" belief given evidence A. The denominator is P(A) from Total Probability law.

Key Identities & Tricks

Inclusion-Exclusion (3 sets)
P(A∪B∪C) = P(A)+P(B)+P(C) − P(A∩B) − P(B∩C) − P(A∩C) + P(A∩B∩C)
Expected Value
E[X] = Σ xᵢ · P(xᵢ) E[aX + b] = a·E[X] + b
Linearity of expectation always holds, even for dependent variables.
Variance
Var(X) = E[X²] − (E[X])² Var(aX+b) = a²·Var(X)
Standard deviation σ = √Var(X)
Derangements (!n)
D(n) = n! · Σ (-1)ᵏ/k! k=0..n ≈ n!/e (for large n)
Permutations where no element appears in its original position. D(1)=0, D(2)=1, D(3)=2, D(4)=9.
Profit & Loss — Formula Sheet Aptitude

Core Definitions

Profit
Profit = SP − CP
CP = Cost Price, SP = Selling Price. Profit when SP > CP.
Loss
Loss = CP − SP
Loss when CP > SP.
Profit %
Profit% = (Profit / CP) × 100
Always calculated on Cost Price unless stated otherwise.
Loss %
Loss% = (Loss / CP) × 100
SP from CP & Profit%
SP = CP × (100 + P%) / 100
CP from SP & Profit%
CP = SP × 100 / (100 + P%)
SP from CP & Loss%
SP = CP × (100 − L%) / 100
CP from SP & Loss%
CP = SP × 100 / (100 − L%)

Marked Price & Discount

Marked Price (MP)
MP = CP × (100 + M%) / 100
Also called List Price or Tag Price.
Discount
Discount = MP × D% / 100 SP = MP × (100 − D%) / 100
Net Profit % (Mark-up then Discount)
Net Profit% = M% − D% − (M%×D%)/100
Single-step formula when CP is base. Example: Mark 25%, discount 10% → 25−10−2.5 = 12.5%
Successive Discounts
Equiv. discount = d₁ + d₂ − (d₁×d₂)/100
Two discounts d₁% then d₂% ≠ (d₁+d₂)%. Always less.

Special Cases

Two Articles: Same SP, One Profit One Loss
Net Result = −(x/10)² %
When each article is sold at ±x% and SP is equal, there is always a loss. E.g., 20% each → loss = 4%.
Dishonest Dealer (False Weight)
Profit% = (True − False)/False × 100
Dealer uses 900g instead of 1000g → Profit = 100/900 × 100 ≈ 11.1%
Break-Even
Break-even: SP = CP → 0% profit
The minimum price to avoid loss.
Selling x at cost of y
Profit% = (y−x)/x × 100 if y>x Loss% = (x−y)/x × 100 if x>y
E.g., "Sell 8 for cost of 10" → profit = 2/8×100 = 25%

Pointers — Common Traps

Pointer Arithmetic

int arr[] = {10, 20, 30, 40};
int *p = arr;          // p → arr[0]
p++;                   // p → arr[1]  (advances by sizeof(int) = 4 bytes)
*(p + 2);              // arr[3] = 40  (p is at index 1, +2 = index 3)
printf("%d", *p);      // 20
  • p++ moves by sizeof(*p) bytes, NOT 1 byte.
  • *(p+n) same as p[n]
  • ++*p increments the value; *p++ dereferences then advances pointer.

Pointer to const vs const pointer

const int *p = &x;   // can't change *p, can change p
int *const p = &x;   // can change *p, can't change p
const int *const p;  // can't change either

Common C++ FAQs for TI Tests

FAQ 1: What does printf("%.3f", 1.0/3) print?

Answer: 0.333 — 3 decimal places, rounds correctly.

FAQ 2: What is the output of sizeof("hello")?

Answer: 6 — 5 chars + 1 null terminator '\0'.

FAQ 3: What happens with int x = 5; printf("%d %d", x++, ++x)?

Answer: Undefined Behavior — modifying x twice between sequence points is UB in C/C++. Do not rely on the output.

FAQ 4: Dangling pointer vs memory leak vs null pointer

  • Dangling pointer: Points to freed/out-of-scope memory. Use after free = UB.
  • Memory leak: Allocated memory never freed. Program runs out of heap.
  • Null pointer: Points to address 0. Dereferencing = segfault.

FAQ 5: What is the output of this recursive function?

int f(int n) {
    if (n <= 0) return 0;
    return n + f(n - 1);
}
// f(5) = 5+4+3+2+1+0 = 15  (triangular number = n(n+1)/2)

FAQ 6: Pass by value vs pass by reference vs pointer

void byVal(int x) { x = 99; }          // caller unchanged
void byRef(int &x) { x = 99; }         // caller changed
void byPtr(int *x) { *x = 99; }        // caller changed (dereference)

FAQ 7: Virtual functions & polymorphism

class Base { public: virtual void f() { cout << "Base"; } };
class Derived : public Base { public: void f() { cout << "Derived"; } };
Base *b = new Derived();
b->f();   // prints "Derived" (runtime polymorphism via vtable)

Without virtual, b->f() would print "Base" (static dispatch).

MCQ Practice — Probability 15 Questions
Q1Probability
A coin is tossed 5 times. What is the probability of getting exactly 3 heads?
B(5, 1/2): ⁵C₃ × (1/2)⁵ = 10/32 = 5/16. Note: options A and C are the same value (5/16 = 10/32), so answer is A/C.
Q2Conditional
In a class, 60% students pass Math, 50% pass Physics. 30% pass both. Given a student passed Math, what is the probability they also passed Physics?
P(Physics | Math) = P(Math ∩ Physics) / P(Math) = 0.30 / 0.60 = 0.5 = 1/2
Q3Bayes
A factory has two machines. Machine A produces 60% of output with 2% defective; Machine B produces 40% with 5% defective. A randomly picked item is defective. What is the probability it came from Machine B?
P(B) = 0.4, P(def|B) = 0.05. P(A) = 0.6, P(def|A) = 0.02.
P(def) = 0.6×0.02 + 0.4×0.05 = 0.012 + 0.020 = 0.032.
P(B|def) = (0.4×0.05)/0.032 = 0.020/0.032 = 10/16 = 5/8.
Wait — let's recompute: 20/32 = 5/8 ≈ 0.625. Answer is 5/8. Select the option closest: 10/17 ≈ 0.588 (not exact). Correct answer = 5/8 (not listed — typical TI trick: verify yourself).
Q4Counting
How many 4-digit numbers can be formed using digits 1–7 (no repetition) that are divisible by 5?
Last digit must be 5 (only 5 in {1-7} is divisible by 5). Remaining 3 digits chosen from 6 (digits 1-7 except 5): 6×5×4 = 120
Q5At Least
A bag has 4 white and 6 black balls. 3 balls are drawn randomly. Find P(at least 2 white).
Total = ¹⁰C₃ = 120.
Exactly 2 white: ⁴C₂×⁶C₁ = 6×6 = 36.
Exactly 3 white: ⁴C₃×⁶C₀ = 4×1 = 4.
P(≥2 white) = (36+4)/120 = 40/120 = 1/3
Q6Dice
Two dice are thrown. What is the probability that the sum is a prime number?
Possible sums: 2–12. Prime sums: 2,3,5,7,11.
Count: 2→1, 3→2, 5→4, 7→6, 11→2 = 15 outcomes out of 36.
P = 15/36 = 5/12
Q7Birthday
3 people are selected. What is the probability that no two share a birthday? (365 days, uniform.)
P(all different) = (365/365) × (364/365) × (363/365) = 365×364×363 / 365³ ≈ 0.9918
Q8Geometric Prob.
A point is chosen at random inside a square of side 4. What is the probability it lies inside the inscribed circle?
Circle radius = 2 (inscribed in side 4). Area circle = π×4 = 4π. Area square = 16.
P = 4π/16 = π/4 ≈ 0.785
Q9Cards
From a deck of 52 cards, 2 are drawn without replacement. P(both are kings)?
P = (4/52)×(3/51) = 12/2652 = 1/221
Q10Expected Value
A game: Roll a die. Win ₹10 if >4, lose ₹5 if ≤4. What is the expected gain per game?
P(>4) = 2/6 = 1/3. P(≤4) = 4/6 = 2/3.
E = (1/3)×10 + (2/3)×(−5) = 10/3 − 10/3 = ₹0. It's a fair game!
Q11Derangement
4 letters and 4 addressed envelopes. Letters are placed randomly. How many ways have no letter in its correct envelope?
D(4) = 4!(1 − 1/1! + 1/2! − 1/3! + 1/4!) = 24(1−1+0.5−1/6+1/24) = 24×(9/24) = 9
Q12TI Style
A student appears in 3 exams. P(pass each) = 0.7, independently. What is the probability of passing exactly 2?
B(3, 0.7), k=2: ³C₂×(0.7)²×(0.3)¹ = 3×0.49×0.3 = 0.441
MCQ Practice — Profit & Loss 10 Questions
P1P&L
A man buys an article for ₹800 and sells it for ₹1000. Find the profit percentage.
Profit = 200. Profit% = 200/800 × 100 = 25%
P2P&L
By selling an item for ₹540, a seller incurs a 10% loss. What is the cost price?
CP = SP × 100/(100−L%) = 540 × 100/90 = ₹600
P3Mark up + Discount
A shopkeeper marks up price by 40% and offers a 20% discount. What is the net profit or loss %?
Net% = M% − D% − (M%×D%)/100 = 40 − 20 − (40×20)/100 = 40−20−8 = 12% profit
P4P&L
Two items are sold at ₹99 each. One at 10% profit, another at 10% loss. What is the overall result?
Same SP with ±10%: loss = (10/10)² % = 1%. Total SP = 198. Total CP = 99×100/110 + 99×100/90 = 90 + 110 = 200. Loss = 200−198 = ₹2
P5Successive Discount
A price list shows ₹5000. Discounts of 20% and 10% are given successively. Final price?
After 20%: 5000×0.8 = 4000. After 10%: 4000×0.9 = ₹3600.
Equivalent single discount = 20+10−2 = 28%: 5000×0.72 = 3600. ✓
P6Selling x for y
A vendor sells 12 mangoes for the cost price of 15 mangoes. Find the profit %.
Sells 12 at CP of 15. Profit per batch = 3 mangoes. Profit% = 3/12 × 100 = 25%
P7Dishonest Dealer
A dealer uses a 900g weight to sell 1 kg. Claiming to sell at cost price, what is his actual profit %?
Profit% = (1000−900)/900 × 100 = 100/900 × 100 = 11.11%
P8P&L
After two successive years of 10% profit each year, what is the effective profit % over 2 years?
Effective = 10+10+(10×10)/100 = 20+1 = 21%. Same formula as successive %.
MCQ Practice — C++ Tricky Questions 12 Questions
C1Pointers
What is the output? 
#include <stdio.h>
int main() {
    int a[] = {10, 20, 30, 40, 50};
    int *p = a + 1;
    printf("%d %d %d", *p, *(p+2), p[1]);
}
p = a+1 → points to a[1]=20. *p=20. *(p+2)=a[3]=40. p[1]=a[2]=30. Output: 20 40 30
C2Bitwise
What does x & (x-1) compute when x=12?
12 = 1100, 11 = 1011. 1100 & 1011 = 1000 = 8.
Note: x & (x-1) clears the lowest set bit. If result == 0, x is a power of 2.
C3sizeof
What is the output? 
struct S { char a; int b; char c; };
printf("%zu", sizeof(struct S));
With padding: char a (1) + 3 padding + int b (4) + char c (1) + 3 padding = 12 bytes. Compiler aligns int to 4-byte boundary.
C4Recursion
What does this function return for n=5? 
int f(int n) {
    if (n == 0) return 1;
    if (n % 2 == 0) return f(n/2) * f(n/2);
    return n * f(n-1);
}
n=5 (odd): 5 × f(4).
n=4 (even): f(2)×f(2).
n=2 (even): f(1)×f(1).
n=1 (odd): 1×f(0)=1.
f(2)=1×1=1. f(4)=1×1=1. f(5)=5×1=5.
Wait — n=5: 5×f(4). f(4)=f(2)×f(2). f(2)=f(1)×f(1)=1. f(4)=1. f(5)=5. Answer: 5. (Option D)
C5Pointers
What is the output? 
#include <stdio.h>
void f(int *p) { *p += 10; p = NULL; }
int main() {
    int x = 5;
    f(&x);
    printf("%d", x);
}
*p += 10 modifies x to 15. Then p = NULL only changes the local copy of the pointer inside f — it does NOT affect x. Output: 15
C6Overflow
What is the output? 
#include <stdio.h>
int main() {
    unsigned char x = 255;
    x++;
    printf("%d", x);
}
Unsigned char max = 255. Adding 1 wraps to 0. Unsigned overflow is defined behavior in C/C++ (wraps modulo 256).
C7Static
What is the output? 
#include <stdio.h>
void counter() {
    static int c = 0;
    c++;
    printf("%d ", c);
}
int main() {
    counter(); counter(); counter();
}
Static variable c is initialized to 0 only once and retains its value across calls. Calls: c=1, c=2, c=3. Output: 1 2 3
C8Strings
What is the output? 
#include <stdio.h>
#include <string.h>
int main() {
    char s[] = "Hello";
    printf("%d %d", strlen(s), sizeof(s));
}
strlen does not count '\0' → 5. sizeof counts the null terminator → 6. Output: 5 6
C9OOP
What is the output? 
#include <iostream>
using namespace std;
class A {
public:
    A() { cout << "A "; }
    ~A() { cout << "~A "; }
};
class B : public A {
public:
    B() { cout << "B "; }
    ~B() { cout << "~B "; }
};
int main() { B b; }
Construction: Base first (A), then Derived (B). Destruction: Derived first (~B), then Base (~A). Output: A B ~B ~A

T1TI Past Year Style
In a tournament, Priya must win 2 consecutive matches to win. She can play max 3 matches.

Option A: vs Strong (p=0.4) → vs Weak (p=0.9) → vs Strong
Option B: vs Weak (p=0.9) → vs Strong (p=0.4) → vs Weak

Which option gives a higher winning probability?
Option A: SWS. Win = W1W2 + L1W2W3 = 0.4×0.9 + 0.6×0.9×0.4 = 0.36 + 0.216 = 0.576
Option B: WSW. Win = W1W2 + L1W2W3 = 0.9×0.4 + 0.1×0.4×0.9 = 0.36 + 0.036 = 0.396
Key insight: Play the strong opponent in the middle (not at both ends) → Option A wins
T2EE
An op-amp integrator has R = 10kΩ and C = 1μF. The input is a square wave of amplitude 2V and period 2ms. What is the approximate peak-to-peak output?
Integrator output per half period: ΔV = V_in × T/(2RC) = 2 × (1×10⁻³)/(10⁴ × 10⁻⁶) = 2 × (10⁻³/10⁻²) = 2 × 0.1 = 0.2 V peak-to-peak
T3C Output
What is the output? 
#include <stdio.h>
int main() {
    int x = 10;
    int y = x++ + ++x;
    printf("%d %d", y, x);
}
Technically undefined behavior in C (modifying x twice without a sequence point). However most compilers evaluate left-to-right: x++(=10, x→11), ++x(x→12, =12). y = 10+12 = 22, x = 12. TI tests often expect 22 12 as the "practical" answer.
T4Probability
A fair die is rolled repeatedly until a 6 appears. What is the expected number of rolls?
Geometric distribution with p = 1/6. E[X] = 1/p = 6
T5EE: MOSFET
For an NMOS in saturation with V_GS = 3V, V_th = 1V, k' = 2mA/V², W/L = 10: what is the drain current I_D?
I_D = ½ × k' × (W/L) × (V_GS − V_th)² = ½ × 2×10⁻³ × 10 × (3−1)²
= ½ × 80×10⁻³ = 40 mA
T6P&L
A trader allows a 15% discount on MP and still earns 19% profit. If MP = ₹140, what is the CP?
SP = 140 × 0.85 = ₹119. Profit 19% → SP = 1.19 × CP. CP = 119/1.19 = ₹100
T7Boolean
Simplify: F = A'BC + AB'C + ABC' + ABC
These are the minterms 3,5,6,7 → majority function. Using K-map or algebra:
ABC + AB'C = AC. ABC + ABC' = AB. ABC + A'BC = BC.
So F = AB + BC + AC (any two of three are 1).
