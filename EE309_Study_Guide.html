1. Architecture Basics

What is a Microprocessor System?

  • ALU - Arithmetic Logic Unit
  • CU - Control Unit (coordinates everything, acts like an FSM)
  • Memory - ROM (long term) + RAM
  • I/O Devices

Together these form the Hardware Architecture.

Von Neumann vs Harvard

Von Neumann

  • Single shared memory for data & instructions
  • Single bus (bottleneck!)
  • Instructions & data accessed one at a time
  • Simpler design, cheaper
  • Slower

Harvard (8051 uses this!)

  • Separate program & data memory
  • Separate buses
  • Instructions & data accessed simultaneously
  • More complex, more expensive
  • Faster
Architecture Comparison
Von Neumann Bottleneck
In Von Neumann, instruction fetch and data access compete for the same bus. Harvard avoids this by using separate buses.

Program Memory (ROM/Flash)

OV is set when A & B have the same sign and the result has a different sign.

4. Addressing Modes (5 Types)

#ModeExampleExplanation
1 Immediate MOV A, #2FH Data is right there in the instruction itself (in program memory). The # symbol is the giveaway.
2 Direct MOV A, 30H The value stored at memory address 30H is moved to A. Creates a copy. Only lower 128B RAM + SFRs.
3 Register MOV A, R7 2 bytes: 1 data + 1 opcode. Register number (3 bits) embedded in opcode itself. Fast!
4 Indirect MOV A, @R0 R0/R1 holds an address; the value at that address goes into A. Only R0, R1 (8-bit) or DPTR (16-bit). Works with both internal & external RAM.
5 Indexed MOVC A, @A+DPTR Base address (DPTR or PC) + index (A) = effective address. Used for Look-Up Tables (LUT). Only works with A. Read-only program memory.
Memory Aid for Addressing Modes
"I Drink Red Indian Ink" = Immediate, Direct, Register, Indirect, Indexed

LUT (Look-Up Table) Example

MOV DPTR, #Table    ; base address
MOV A, #index_val   ; index
MOVC A, @A+DPTR     ; fetch value from table
...
Table: DB 10h, 20h, 30h, 40h  ; data bytes

DPTR is used when LUT is far from instruction. If LUT is right after the instruction, PC naturally becomes the base (use MOVC A, @A+PC).

This is a classic exam trap. INC does NOT set carry, overflow, or any flag. Only ADD/ADDC/SUBB do.
DA (Decimal Adjust) - How it works

After adding two BCD numbers with ADD/ADDC:

  • If lower nibble > 9 OR AC=1, add 06H to lower nibble
  • If upper nibble > 9 OR CY=1, add 60H to upper nibble

Example: 45h + 45h = 8Ah. DA sees lower nibble A>9, adds 6 → result = 90h (BCD for 90).

6. Branching Instructions: JMP vs CALL

The Key Difference

JMP (Just Jump)

Simply changes PC to new location. Nothing pushed to stack. No RET required.

CALL (Call Subroutine)

Pushes return address (PC) onto stack, then jumps. Clubbed with RET to return.

Exam Classic - ACALL Page Address

ACALL uses 11 address bits from the instruction + 5 page bits from PC_next (not current PC!).

Why PC_next? Because instruction fetch happens first → PC increments → then the upper 5 bits are taken from this new PC value.

This means ACALL can only call within the same 2KB page as the next instruction.

Read vs Write vs Read-Modify-Write

  • Read (from pins): MOV A, P0 - reads actual pin values. Does NOT change port SFR.
  • Write (to latch): MOV P0, #55H - writes to port SFR/latch. Changes output.
  • Read-Modify-Write: ANL P0, A or DJNZ P1, label - reads from SFR (latch), modifies, writes back. Reads entire byte, modifies target bits, writes whole byte back.
To use a port as input
You must write '1' to the port latch first! MOV P0, #FFH then read. Output port unless we specifically write '1' to the port/pin.

8. Timers/Counters - All 4 Modes

8051 has two 16-bit timers: Timer 0 (TH0 + TL0) and Timer 1 (TH1 + TL1).

Each can work as a Timer (counts internal clock / machine cycles) or as a Counter (counts external edges on T0/T1 pins).

How the Timer Clock Works

Timer Clock

Timer mode: counts at fosc / 12 (= 1 count per machine cycle)

Counter mode: counts falling edges on external pin (T0 = P3.4, T1 = P3.5)

The edge is sampled in S5P2 of each machine cycle, so max external count rate = fosc/24

NOT bit addressable! Must be written as a whole byte.

Mode 1: 16-bit Timer (most common for delays)

This is the workhorse mode

Full 16-bit counter: TLx (low byte) + THx (high byte). Counts from loaded value up to FFFFH, then overflows to 0000H and sets TFx = 1.

Max count: 216 = 65,536

No auto-reload! After overflow, you must manually reload THx and TLx in your ISR.

Delay Calculation - Mode 1
Delay = (65536 - initial_value) x (12 / f_osc)

To get a specific count N:
  Load value = 65536 - N

Example: f_osc = 11.0592 MHz, want 1000 counts
  Load = 65536 - 1000 = 64536 = FC18H
  TH0 = FCH, TL0 = 18H
  Delay = 1000 x (12/11059200) = 1.085 ms

Mode 2: 8-bit Auto-Reload (best for baud rate generation)

Key Mode for Serial Communication

TLx counts up. When it overflows (FFH → 00H), TFx is set AND the value in THx is automatically reloaded into TLx.

THx holds the reload value (you set it once, it keeps reloading).

Max count: 256 (8-bit). Good for generating precise, repetitive time intervals.

Most commonly used: Timer 1 in Mode 2 for generating serial port baud rate.

Mode 3: Split Timer (Timer 0 only!)

Splits Timer 0 into two independent 8-bit timers:

  • TL0: Uses C/T, GATE, TR0, TF0 (like a normal 8-bit timer 0)
  • TH0: Counts machine cycles only (timer mode). Uses TR1 and TF1 (borrows Timer 1's control bits!)

When Timer 0 is in Mode 3, Timer 1 can still run in Modes 0/1/2 but cannot set TF1 (it's being used by TH0). Timer 1 is usually used for baud rate generation in this case.

9. Interrupts & Priority System

What Happens When an Interrupt Fires

ISR Entry - What Hardware Does Automatically
  1. Finishes the current instruction (however many MCs it needs)
  2. Pushes PC onto stack (return address)
  3. Loads PC with the interrupt vector address

Hardware does NOT save: Registers, PSW, ACC, or anything else! Only PC is saved. YOU must PUSH/POP them in your ISR.

ISR Structure (What YOU must write)

; ISR Template
  PUSH ACC        ; Save registers that ISR will use
  PUSH PSW
  PUSH <other regs>

  ; ... your ISR code ...

  POP <other regs>  ; Restore in reverse order
  POP PSW
  POP ACC
  RETI              ; Return from interrupt (NOT RET!)
RETI vs RET

RETI does two things: (1) pops PC from stack like RET, and (2) clears the interrupt priority flags so the system knows this priority level is free again. Using plain RET would mess up the priority system!

IP Register - Interrupt Priority (B8H, bit addressable)

Critical Concept - Two-Level Priority

Each interrupt can be set to HIGH (bit=1) or LOW (bit=0) priority. That's it - only 2 levels.

IP does NOT change the order within a level! It only selects which group (high or low) an interrupt belongs to.

Within each group, the default polling order (IE0 > TF0 > IE1 > TF1 > RI/TI) STILL applies.

10. Serial Communication (UART)

Synchronous vs Asynchronous

Synchronous

  • Clock signal sent alongside data
  • Data sent in blocks of bytes with preamble
  • No start/stop bits needed
  • More reliable

Asynchronous (UART)

  • No shared clock line
  • Uses start bit + data + stop bit for framing
  • Both sides must agree on same baud rate
  • 8051 uses this (modes 1,2,3)

Baud Rate Calculation

The Master Formula (Modes 1 & 3)
Baud Rate = (2^SMOD / 32) x Timer 1 overflow rate

Timer 1 overflow rate = f_osc / (12 x (256 - TH1))

Combined:
Baud Rate = (2^SMOD / 32) x f_osc / (12 x (256 - TH1))

SMOD = bit 7 of PCON register
  SMOD = 0: normal
  SMOD = 1: doubles baud rate
Why 11.0592 MHz and NOT 12 MHz?

12 MHz cannot generate standard baud rates! 11.0592 MHz divides evenly to give exact standard rates like 9600, 19200, etc.

With 11.0592 MHz, SMOD=1, TH1 = FDH (253):
Baud = (2/32) x 11059200 / (12 x (256-253))
     = (1/16) x 11059200 / 36
     = 19200 bps       (If SMOD=0: 9600 bps)

Data Transfer Rate Concepts

Three Ways to Measure Speed
  • Baud Rate: Number of signal changes per second (Hz). Used in modems.
  • Data Rate: Number of bits transferred per second (bps). If each signal has N voltage levels, data rate = baud rate x log2(N).
  • Effective Data Rate: Number of actual data bits per second (excluding start/stop/parity overhead).

Example: 2400 baud, 4 voltage levels, 1 start + 8 data + 2 stop bits:

Data rate = 2400 x log2(4) = 2400 x 2 = 4800 bps
Effective ratio = 8 / (1+8+2) = 8/11
Effective data rate = 4800 x 8/11 = 3490.9 bps

Transmit & Receive Process

Transmit

  1. Write to SBUF: MOV SBUF, A
  2. Hardware sends start bit, 8 data bits, stop bit
  3. When done, TI flag is set
  4. You must clear TI: CLR TI

Receive

  1. Set REN = 1 to enable receiving
  2. Hardware detects start bit, shifts in data
  3. When byte complete, RI flag is set
  4. Read SBUF: MOV A, SBUF
  5. Clear RI: CLR RI
RI and TI share the SAME interrupt vector (0023H)!

Your serial ISR must check which flag is set to determine if it was a transmit-complete or receive-complete event.

11. Multiprocessor Communication

The Big Idea

One Master 8051 needs to communicate with multiple Slave 8051s on a shared serial bus. Problem: How does the master address a specific slave without every slave processing every byte?

Solution: Use the 9th data bit (TB8/RB8) in Mode 2 or Mode 3 as an address/data flag, combined with the SM2 bit in SCON.

How SM2 Filtering Works

The SM2 Mechanism

When SM2 = 1 in a slave: the RI flag is only set when the received 9th bit = 1. Frames with 9th bit = 0 are silently ignored (RI never set, no interrupt).

When SM2 = 0: RI is set for every received frame, regardless of the 9th bit.

; ISR at 0023H:
PUSH ACC

CLR RI
MOV A, SBUF
; Check if address matches
CJNE A, #02H, NOT_ME

SJMP DONE

SJMP DONE

DONE:
POP ACC
RETI
Exam Tip - Multiprocessor

Key things to remember: (1) Must use Mode 2 or Mode 3 (need 9th bit). (2) SM2=1 means "only interrupt me for address frames". (3) Master always has SM2=0. (4) Selected slave clears SM2 to receive data, then restores SM2=1 when done.

Key Connection: 8051 vs MIPS

8051 = Accumulator-based CISC. Most operations go through ACC. Variable instruction sizes (1-3 bytes).

MIPS = Register-Register (Load-Store) RISC. 32 GPRs, only lw/sw touch memory. Fixed 32-bit instructions.

What Makes a Good ISA?

  • Require less program memory (avoid rarely used instructions)
  • Be easier to compile (less optimization needed)
  • Involve easier processor design (smaller core, simpler circuits)
  • Provide faster program execution

ISA depends on: operations supported by ALU, operand location (register/memory/stack), number of operands (fixed or variable), operand range, and instruction size.

CPU Performance Equation

The Iron Law of Performance
CPU Time = Instruction Count x CPI x Clock Cycle Time

         = (IC x CPI) / Clock Rate

Where:
  IC  = Instruction Count (determined by ISA & compiler)
  CPI = Cycles Per Instruction (determined by CPU design)
  Clock Rate = f_clk (determined by hardware technology)

What Affects Each Factor?

FactorDetermined ByImprove By
ICISA, compiler, algorithmBetter algorithm, compiler optimization
CPICPU microarchitecturePipelining, caching, branch prediction
Clock RateTechnology, circuit designBetter fabrication, shorter critical path

Throughput vs Latency

Throughput

= #tasks / total time

How many tasks completed per second. Pipelining increases throughput!

Latency

= time for one task to complete

How long one task takes from start to finish. Pipelining does NOT improve latency.

Amdahl's Law

Amdahl's Law - The Speedup Limit
Speedup = 1 / [(1 - f) + f/S]

Where:
  f = fraction of execution time that CAN be improved
  S = speedup of the improved portion

Key Insight: Even with infinite speedup of f:
  Max Speedup = 1 / (1 - f)

Example: 80% of code is parallelizable, 4x speedup on that part:
  Speedup = 1 / [(1-0.8) + 0.8/4] = 1 / [0.2 + 0.2] = 2.5x
  (NOT 4x! The 20% serial portion limits us!)
Amdahl's Law - Visual Intuition

Power, Energy & Vdd Tradeoffs

Why Power Matters

Power determines heat dissipation (cooling cost), battery life (mobile devices), and ultimately limits how fast we can clock a chip. The "power wall" is a major reason clock speeds stopped increasing around 2005.

Two Types of Power Dissipation
Total Power = Dynamic Power + Static Power

Dynamic Power (switching activity):
  P_dynamic = C x V_dd^2 x f_clk x alpha

  C     = load capacitance (determined by circuit/technology)
  V_dd  = supply voltage (e.g., 1.0V, 1.2V, 3.3V, 5V)
  f_clk = clock frequency
  alpha = activity factor (fraction of gates switching per cycle)

Static Power (leakage - always on!):
  P_static = V_dd x I_leakage

  Leakage current flows even when transistors are "off"
  Becomes dominant at smaller technology nodes (45nm, 7nm, etc.)
  Cannot be reduced by lowering frequency!
Energy & Performance Products
Energy per operation:   E = C x V_dd^2     (Joules per switching event)
Delay (gate delay):     t_d ~ V_dd / (V_dd - V_th)^2
                        (approximately ~ 1/V_dd for V_dd >> V_th)

Power-Delay Product:    PDP ~ C x V_dd^3     (lower = more efficient)
Energy-Delay Product:   EDP ~ C x V_dd       (lower = better metric)

Key Insight: Reducing V_dd improves energy efficiency but
             increases delay (slower operation).

8051 operates at Vcc = 5V (or 3.3V for low-power variants). Power consumption is in the milliwatt range.

Hardware Parallelism for Low Power

Trick: Duplicate hardware, run each copy at half speed → same throughput but significantly lower power.

Why? If you halve the frequency, you can also reduce Vdd (slower transistors tolerate lower voltage). Since P ~ V2 × f:

Original:    P = C x V^2 x f
2 copies at half speed, V reduced to 0.7V:
Each copy:   P = C x (0.7V)^2 x (f/2) = 0.245 x C x V^2 x f
Total (2x):  P_total = 0.49 x C x V^2 x f  (51% power reduction!)

Same throughput, half the power. This is why modern CPUs have many cores at lower clock speeds.

Which Metric to Optimize?

  --> Focuses entirely on energy consumption/task
  --> For extremely low energy use cases (sensors, IoT)

  --> Optimized to balance Energy and Performance
  --> Has a minimum at an optimal V_dd (~0.9V in the lecture graph)

Dynamic Voltage & Frequency Scaling (DVFS)

How Modern Processors Save Power

DVFS dynamically adjusts Vdd and fclk based on workload:

  • High workload: Increase Vdd and frequency for max performance
  • Low workload: Decrease Vdd and frequency to save power
  • Idle: Drop to minimum voltage, or even power gate (turn off) parts of the chip

The OS/firmware controls this. Reduces Vdd → power drops quadratically. This is why your laptop doesn't always run at max frequency.

Exam Tips - Power & Vcc
  • Dynamic power dominates at higher voltages and frequencies
  • Static power (leakage) dominates at very low voltages / advanced nodes
  • Reducing Vdd by 2x reduces dynamic power by 4x (quadratic!)
  • Reducing frequency by 2x reduces dynamic power by 2x (linear)
  • PDP ~ CVdd2 (energy/task, quadratic dependence on Vdd)
  • EDP has a minimum — there's an optimal Vdd that minimizes energy-delay
  • Energy-Delay tradeoff can be broken using more hardware (parallelism, pipelining)
  • 8051 Vcc = 5V (40-pin DIP: pin 40 = Vcc, pin 20 = GND)
  • Performance benchmarks differ by task: gaming, video encoding, compiler, AI training

13. MIPS ISA - Complete Guide

What is MIPS?

MIPS = Microprocessor without Interlocked Pipeline Stages. A classic RISC architecture designed for efficient pipelining.

MiniMIPS = The subset used in EE309 (from Parhami textbook). A teaching-friendly version of full MIPS.

MIPS Design Principles

  1. Simplicity favors regularity - 3 operands per instruction, fixed formats
  2. Smaller is faster - Only 32 registers (5 bits to address)
  3. Make the common case fast - Immediate operands (I-type) for common operations
  4. Good design demands good compromises - Same length instructions with different formats

Loading a 32-bit Constant

MIPS immediate fields are only 16 bits. To load a full 32-bit constant:

Cannot use addi for lower half because sign-extension may corrupt upper bits!

MIPS Addressing Summary

  • Big-endian: Most significant byte at lowest address
  • Word = 4 bytes: Addresses are byte addresses, words at 0, 4, 8, 12...
  • Word-aligned: Word addresses must be divisible by 4
  • Memory size: 230 words (232 bytes) addressable

14. MIPS Datapath: Single-Cycle & Multi-Cycle

Single-Cycle Datapath

Core Idea

Every instruction completes in exactly one clock cycle. CPI = 1 always.

Problem: Clock cycle must be long enough for the slowest instruction (typically lw). Fast instructions (like add) waste time.

Single-Cycle Instruction Execution Flow

How Each Instruction Type Uses the Datapath

Multi-Cycle Datapath

Why Multi-Cycle?

Single-cycle wastes time: clock = slowest instruction. Multi-cycle breaks each instruction into smaller steps, each taking one (shorter) clock cycle. Different instructions take different numbers of cycles (3-5).

Advantages: Shorter clock period + resource reuse (single ALU, single memory).

Key Differences from Single-Cycle

  • Single memory unit for both instructions and data (not separate caches)
  • Single ALU (no separate adder for PC+4)
  • Intermediate registers between stages: IR (Instruction Register), MDR (Memory Data Register), X, Y, Z registers - hold values between cycles
  • These intermediate registers are NOT visible to the programmer

Multi-Cycle Execution: Cycle by Cycle

Cycle 1: Instruction Fetch (ALL instructions)
IR ← Memory[PC]     ; Fetch instruction, store in IR
PC ← PC + 4          ; Increment PC
Cycle 2: Instruction Decode & Register Fetch (ALL instructions)
X ← Reg[IR[25:21]]   ; Read register rs
Y ← Reg[IR[20:16]]   ; Read register rt (even if not needed)
Z ← PC + SE(IR[15:0]) << 2   ; Compute branch target (speculative)
Cycle 3: Execution (depends on instruction type)
Memory reference (lw/sw):  Z ← X + SE(IR[15:0])     ; Compute address
ALU (R-type):              Z ← X op Y               ; Perform operation
Branch (beq/bne):          if(X == Y) PC ← Z        ; Conditional branch
Jump (j):                  PC ← {PC[31:28], IR[25:0], 00}
Cycle 4: Memory Access or R-type Completion
lw:      MDR ← Memory[Z]        ; Read from data memory
sw:      Memory[Z] ← Y            ; Write rt to data memory
R-type:  Reg[IR[15:11]] ← Z      ; Write ALU result to rd
Cycle 5: Write Back (lw only)
lw:      Reg[IR[20:16]] ← MDR   ; Write memory data to rt

15. Pipelining: Stages, Hazards & Performance

The Core Insight

Pipelining is like a laundry assembly line: while one load is drying, start washing the next. Each instruction still takes the same time (latency), but throughput increases because multiple instructions overlap.

Ideally, a 5-stage pipeline gives 5x throughput improvement (but NOT 5x speedup on a single instruction).

MIPS 5-Stage Pipeline

The 5 Pipeline Stages

Pipeline Timing Diagram

Each instruction enters the pipeline one cycle after the previous one:

InstructionCC1CC2CC3CC4CC5CC6CC7CC8CC9
lw $s1, 8($t0) IFIDEXMEMWB
lw $s2, 16($t0) IFIDEXMEMWB
lw $s3, 24($t0) IFIDEXMEMWB
add $t1, $s1, $s2 IFIDEXMEMWB
sub $t2, $s1, $s3 IFIDEXMEMWB

Start-up Region

First few cycles where pipeline isn't fully utilized. All stages are not busy yet.

Drainage Region

Last few cycles where earlier stages are idle because last instruction has moved on.

Pipeline Performance

Performance Comparison
Single-cycle: N instructions take N x T_longest cycles

Pipelined: N instructions take (N + stages - 1) x T_stage cycles
  where T_stage = T_longest_stage (not T_longest_instruction!)

Example: IF=2ns, ID=1ns, EX=2ns, MEM=2ns, WB=1ns
  Single-cycle clock = 2+1+2+2+1 = 8 ns
  Pipelined clock = max(2,1,2,2,1) = 2 ns per stage

  3 instructions:
  Single-cycle: 3 x 8 = 24 ns
  Pipelined: (3 + 5 - 1) x 2 = 14 ns

Speedup approaches number of stages as N → large

Why MIPS is Pipeline-Friendly

  • Fixed-length instructions - fetch in 1 stage, decode in next
  • Few instruction formats - register fields always in same position
  • Memory operands only in lw/sw - address calc in EX, access in MEM
  • Word-aligned memory - single transfer per access
  • Each functional unit used once per instruction at the same stage

Pipeline Hazards

Situations where the next instruction cannot execute in the next clock cycle.

Data Hazard

An instruction depends on the result of a previous instruction still in the pipeline.

add $s0, $t0, $t1
sub $t2, $s0, $t3
     ^^ needs $s0
     but add hasn't
     written it yet!

Solutions: Forwarding/bypassing, stalling (NOP insertion)

Control Hazard

Branch decision not known when next instruction needs to be fetched.

beq $s0, $s1, L
add $t0, ...  ; fetch this?
              ; or the
              ; branch target?

Solutions: Branch prediction, delayed branching, stall

Structural Hazard

Two instructions need the same hardware resource simultaneously.

Example: single memory for both inst fetch and data access.

Solution: Duplicate resources (separate inst & data caches)

Key Pipeline Observations
  • Pipelining improves throughput, not latency of individual instructions
  • Pipeline clock rate limited by slowest stage
  • Pipeline registers (IF/ID, ID/EX, etc.) add small overhead
  • In MIPS, register file can be read in first half and written in second half of the same cycle (solves some data hazards)

16. Cache Memory: Hierarchy, Mapping & Performance

The Memory Wall Problem

Why We Need Cache

Processor speed: ~1 ns per operation. Main memory (DRAM) access: ~50-100 ns. That's a 50-100x gap!

Solution: Place a small, fast cache (SRAM) between CPU and main memory. Cache holds copies of frequently used data.

Memory Hierarchy

Memory Hierarchy Pyramid

Principle of Locality

Temporal Locality

If a data item is accessed, it will likely be accessed again soon.

Example: Loop variables, frequently called functions.

Cache exploits this by keeping recently used data.

Spatial Locality

If a data item is accessed, nearby items will likely be accessed soon.

Example: Array elements, sequential instructions.

Cache exploits this by fetching entire blocks (cache lines).

Cache Terminology

TermDefinition
Block (Line)Unit of data transferred between cache and memory (e.g., 16 or 32 bytes)
Cache HitRequested data found in cache. Access time = hit time.
Cache MissData NOT in cache. Must fetch from main memory. Time = hit time + miss penalty.
Hit RateFraction of accesses found in cache (typically 90-99%)
Miss Rate= 1 - Hit Rate
Miss PenaltyTime to replace a block + deliver to CPU (10-100+ cycles)
Valid Bit1 = cache entry contains valid data; 0 = empty/invalid
Average Memory Access Time (AMAT)
AMAT = Hit Time + (Miss Rate x Miss Penalty)

Example: Hit time = 1 cycle, miss rate = 5%, miss penalty = 100 cycles
  AMAT = 1 + (0.05 x 100) = 1 + 5 = 6 cycles

Cache Organization: Three Types

1. Direct-Mapped Cache

Each memory block maps to exactly one cache location.

Cache block index = (Block # in memory) mod (# blocks in cache)

Direct-Mapped Cache Address Breakdown

Lookup: Use Index to find the cache block → compare stored Tag with address Tag → check Valid bit = 1 → Hit!

Pros: Simple, fast lookup. Cons: Conflict misses (two blocks mapping to same index evict each other).

2. Fully Associative Cache

A block can be placed in ANY cache location. No index needed!

Address = Tag | Byte Offset only.

Must compare tag against all entries simultaneously (needs comparators for every entry).

Pros: No conflict misses. Cons: Expensive hardware (many comparators), slow for large caches.

3. N-Way Set-Associative Cache

The Practical Compromise

Cache divided into sets. Each set has N blocks (N "ways").

Set index = (Block #) mod (# of sets)

Within a set, block can go in any of the N ways (fully associative within the set).

Set-Associative Address Breakdown

Cache Organization Comparison

TypeMappingLookup SpeedConflict MissesHardware Cost
Direct-Mapped1 possible locationFastestHighCheapest
N-way Set-AssocN possible locationsModerateModerateModerate
Fully AssociativeAny locationSlowestNoneMost expensive
Cache Address Bit Calculation
Given: Cache size = S bytes, Block size = B bytes, N-way set-associative

Number of blocks = S / B
Number of sets   = (S / B) / N

Offset bits = log2(B)          ; to address bytes within a block
Index bits  = log2(#sets)      ; to select which set
Tag bits    = 32 - index - offset  ; remaining bits

Example: 128B cache, 16B blocks, 2-way set-associative
  Blocks = 128/16 = 8
  Sets = 8/2 = 4
  Offset = log2(16) = 4 bits
  Index = log2(4) = 2 bits
  Tag = 32 - 2 - 4 = 26 bits

Split Cache

Most processors use separate instruction cache and data cache (split cache / Harvard-style at L1).

This avoids structural hazards in pipelining (IF and MEM stages can access memory simultaneously).
